The CYBERCORP Forensic Minimal (FP-Master) platform is an integrated SOC/CERT ecosystem for forensic analysis, hunting, incident response and threat intelligence — deployed in Docker containers behind a single Nginx TLS reverse proxy.

Mission & scope

This minimal edition focuses operational capabilities on: evidence ingestion, SIEM search (OpenSearch), HELK hunting (Sysmon/Linux/Zeek + Sigma/MITRE), Velociraptor DFIR collection, Timesketch timelines, CTI stack (OpenCTI, MISP, TheHive, Cortex) and Grafana supervision. CERT and IT portals unify analyst access without multiplying URLs.

Functional modules

CERT portal

SOC overview, 11+ service health, ingest, master incidents, HELK hunting, VR DFIR, CTI, administration. File: portal-cert/server.js.

IT portal

Evidence upload via CERT token, operation tracking, HELK/VR lab sync. URL: /it/.

Ingestion

Python worker: EVTX, Plaso, CSV, syslog, STIX → OpenSearch + Timesketch + MinIO. Redis queue.

OpenSearch

2-node cluster, Dashboards, Logstash/Filebeat, forensic-*, fp-* indices, HELK/VR sync.

HELK sidecar

Logstash pipelines 0010–0099, Kibana hunting, Grafana helk-*, bridge sync/export.

Velociraptor

GUI + collection bridge, ForensicFull artefacts, offline lab playbooks, OS/TS/CERT export.

Timesketch

Chronological investigation, auto ingest import, HELK/VR exports.

CTI & IR

OpenCTI STIX hub, MISP IOC, TheHive cases, Cortex analyzers, portal webhook.

Observability

Grafana, Prometheus, Loki — fp-platform-health-gf dashboard, portal heatmap.

Ingest limits (portal)

  • Maximum 100 files per upload session
  • Max size 5.4 GB per file
  • Extensions: EVTX, EVT, LOG, syslog, Plaso, CSV, JSONL, PCAP, STIX, ZIP/KAPE…
  • Send to HELK option on CERT upload to feed hunting
