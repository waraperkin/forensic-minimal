Full architecture of the CYBERCORP Forensic Minimal platform: single Nginx TLS entry point, CERT/IT portals, OpenSearch cluster, HELK and Velociraptor sidecars, CTI/IR stack and observability. Docker networks: forensic-net, helk_net, velociraptor_net.

    Portal->>MinIO: PutObject (bucket by type)
    Portal->>Redis: enqueueIngestJob()
    Worker->>Redis: dequeue job
    Worker->>MinIO: GetObject
    Worker->>Worker: Parse EVTX / CSV / Plaso / text
    Worker->>OS: bulk index forensic-* / fp-*
    Worker->>TI: IOC enrichment (optional)
    Worker->>TS: timesketch_io.py (9-col CSV)
    Worker-->>Portal: ACK / error → ingest-errors

10. Security & operations

  • TLS terminated on Nginx — HSTS enabled, certs in config/nginx/certs/
  • Portal session — HttpOnly cookie; optional TOTP MFA
  • IT portal — access only via time/usage-limited tokens
  • Demo mode (Documentation) — blocks purge/delete/regenerate
  • Maintenance — ./forensic.sh check-health, docker compose logs cert-portal
