Operational security for the portal and platform: authentication, sessions, IT token access, audit and training mode.

IT portal — token model

  • CERT analyst generates token via POST /api/tokens/generate (case_id, expiration, max uses)
  • IT team accesses /it/?token=… without permanent account
  • Upload limited to token scope; traceability in forensic-tokens*
  • Revocation: DELETE /api/tokens/:id

TLS & network

  • TLS terminated on Nginx — certificates in config/nginx/certs/
  • Strict-Transport-Security header enabled
  • SSL fingerprint displayed in IT Health for remote team verification
  • Internal services (OpenSearch, Redis) not exposed outside Docker network

Demo mode (training)

Enable from portal Documentation:

  • Blocks delete, purge, regenerate actions
  • Implemented in portal-doc.js (blockDemoDestructive)
  • Useful for demonstrations without destructive write risk

Analyst best practices

  1. Never share Access Center credentials over insecure channels
  2. Review all IOC/report exports before external send
  3. Use short-lived IT tokens for urgent uploads
  4. Verify demo mode before a training session
