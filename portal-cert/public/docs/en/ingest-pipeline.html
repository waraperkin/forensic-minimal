Detailed steps

  1. Reception — portal-cert/server.js or portal-it/server.js receives files (multer). Metadata: case_id, analyst, priority, source OS.
  2. Validation — lib/upload-limits.js: allowed extensions, max 5.4 GB, max 100 files.
  3. Storage — MinIO bucket by type (logs-windows, timesketch, pcap, iocs, kape…).
  4. Enqueue — lib/ingest-queue.js pushes JSON job to Redis.
  5. Processing — ingest-worker/worker.py consumes queue, parses and bulk-indexes.
  6. Enrichment — ti_enrichment.py may enrich with OpenCTI/MISP context.
  7. Timesketch — timesketch_pipeline.py + strict 9-column CSV.
  8. HELK (optional) — if checked on upload, lib/helk-connector.js POST to HELK Logstash.

Logstash & Filebeat (network flow)

In parallel with portal upload, FP Logstash listens on:

  • :5044 — Beats (agents/Filebeat)
  • :5045 — JSON lines
  • :5140/udp — Syslog
  • :5555 — HEC (HTTP Event Collector)

Pipelines: config/logstash/pipeline/00-inputs.conf, 10-windows.conf, 20-linux-macos.conf, 40-multi-source.conf. Filebeat collects Docker container logs to OpenSearch.
