Portail IT — modèle token

  • L'analyste CERT génère un token via POST /api/tokens/generate (case_id, expiration, max utilisations)
  • L'équipe IT accède à /it/?token=… sans compte permanent
  • Upload limité au périmètre du token ; traçabilité dans forensic-tokens*
  • Révocation : DELETE /api/tokens/:id

TLS & réseau

  • TLS terminé sur Nginx — certificats dans config/nginx/certs/
  • Header Strict-Transport-Security activé
  • Fingerprint SSL affiché dans Santé IT pour vérification équipes distantes
  • Services internes (OpenSearch, Redis) non exposés hors réseau Docker

Mode démo (formation)

Activable depuis Documentation portail :

  • Bloque les actions delete, purge, regenerate
  • Implémenté dans portal-doc.js (blockDemoDestructive)
  • Utile pour démonstrations sans risque d'écriture destructive

Bonnes pratiques analyste

  1. Ne jamais partager les credentials du Centre d'accès hors canal sécurisé
  2. Relire tout export IOC/rapport avant envoi externe
  3. Utiliser des tokens IT à expiration courte pour les dépôts urgents
  4. Vérifier le mode démo avant une session de formation
