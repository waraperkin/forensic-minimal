Étapes détaillées

  1. Réception — portal-cert/server.js ou portal-it/server.js reçoit les fichiers (multer). Métadonnées : case_id, analyste, priorité, OS source.
  2. Validation — lib/upload-limits.js : extensions autorisées, taille max 5.4 Go, nombre max 100 fichiers.
  3. Stockage — MinIO bucket selon type (logs-windows, timesketch, pcap, iocs, kape…).
  4. Enqueue — lib/ingest-queue.js pousse un job JSON dans Redis.
  5. Traitement — ingest-worker/worker.py consomme la queue, parse et bulk-indexe.
  6. Enrichissement — ti_enrichment.py peut enrichir avec contexte OpenCTI/MISP.
  7. Timesketch — timesketch_pipeline.py + CSV 9 colonnes strictes.
  8. HELK (option) — si coché à l'upload, lib/helk-connector.js POST vers Logstash HELK.

Logstash & Filebeat (flux réseau)

En parallèle de l'upload portail, Logstash FP écoute :

  • :5044 — Beats (agents/Filebeat)
  • :5045 — JSON lines
  • :5140/udp — Syslog
  • :5555 — HEC (HTTP Event Collector)

Pipelines : config/logstash/pipeline/00-inputs.conf, 10-windows.conf, 20-linux-macos.conf, 40-multi-source.conf. Filebeat collecte les logs conteneurs Docker vers OpenSearch.
