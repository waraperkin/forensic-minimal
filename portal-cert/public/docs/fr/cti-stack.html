Connecteurs OpenCTI

Définis dans docker-compose.opencti.yml :

  • MITRE ATT&CK — matrices et techniques
  • CVE — flux vulnérabilités
  • OpenCTI datasets — jeux de données officiels
  • Connecteurs tiers : AbuseIPDB, AlienVault, MalwareBazaar, URLhaus, ThreatFox, CISA KEV, Shodan…

Activation : scripts/opencti_master_setup.py lors du ./forensic.sh full-start.

Workflow IR type

  1. Détection hunting HELK ou alerte ingest → création incident master
  2. Extraction IOC → export /api/helk/export-cti ou saisie manuelle OpenCTI
  3. Création cas TheHive, ajout observables
  4. Lancement analyseurs Cortex (hash, URL, IP)
  5. Partage IOC pertinents vers MISP (communauté / interne)
  6. Documentation dans KB portail + clôture cas

Config : config/opencti/ti-turbo.env, config/thehive/application.conf, config/cortex/application.conf. Doc : docs/PORTAL/CTI.md, docs/PORTAL/IR.md.
