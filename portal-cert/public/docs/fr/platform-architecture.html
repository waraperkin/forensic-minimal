3. Portail CERT — architecture applicative

Navigateur
   │
   ├─► HTML/JS/CSS statiques (portal-shared → /shared/)
   │
   ├─► /api/auth/*        Session cookie, MFA TOTP, rôles admin/analyste
   ├─► /api/overview/*    Santé services, ingest stats, CTI summary
   ├─► /api/master/*      Incidents, KB, dashboards FP-Master
   ├─► /api/upload        Dépôt evidence → MinIO + queue Redis
   ├─► /api/audit/events  Journal d'activité (modifications plateforme)
   │
   └─► /api/threat/*      Proxy Node → control planes
         ├─► sekoia-controlplane:8081   (intakes, règles, fetch, CC)
         └─► sentinelone-controlplane:8082 (agents, policies, DV)

Chargement JS modulaire via portal-lazy.js : threat-platforms, governance, cert-tools, sekoia-cc, portal-doc, portal-ai.

4. Pipeline d'ingestion forensique

Upload CERT/IT (multer)
      │
      ├─► MinIO (bucket par type: logs-windows, timesketch, pcap, iocs…)
      ├─► Redis LPUSH ingest-queue
      │
      ▼
 ingest-worker (Python)
      ├─ Parse EVTX / texte / CSV / Plaso
      ├─ Bulk index OpenSearch (forensic-*, fp-threat-*)
      ├─ timesketch_pipeline → CSV 9 colonnes → Timesketch API
      └─ Métadonnées ingest → /api/master/ingest

Types supportés : EVTX, EVT, LOG, syslog, Plaso, CSV, JSONL, PCAP, STIX, ZIP/KAPE… Voir onglet Upload pour limites (100 fichiers, 5.4 Go/fichier).

Logstash (ports 5044 Beats, 5045 JSON, 5140 syslog, 5555 HEC) alimente également OpenSearch pour les flux réseau/SIEM.

12. Flux de corrélation analyste (cas d'usage)

  1. Alerte Sekoia / incident FP-Master → fiche dans portail CERT
  2. Collecte ciblée Sekoia fetch ou S1 DV sur l'asset
  3. Investigation asset / XDR View — corrélation multi-source
  4. Timeline Builder → export Timesketch
  5. IOC → corrélation OpenCTI/MISP + règles
  6. Centre d'audit — traçabilité des modifications

13. Sécurité réseau & session

  • TLS terminé sur Nginx ; HSTS activé
  • Session portail : cookie HttpOnly, MFA TOTP optionnel
  • Portail IT : accès par token à usage limité (dépôt preuves)
  • API threat : Bearer JWT Sekoia (durée ~15 min) saisi dans Configuration
  • Mode démo documentation : bloque purge/delete/regenerate
