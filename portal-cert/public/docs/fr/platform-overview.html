La plateforme CYBERCORP Forensic Minimal (FP-Master) est un écosystème SOC/CERT intégré pour l'analyse forensique, le hunting, la réponse à incident et le renseignement sur les menaces — déployé en conteneurs Docker derrière un reverse proxy Nginx TLS unique.

Mission & périmètre

Cette version minimaliste concentre les capacités opérationnelles sur : ingestion de preuves, recherche SIEM (OpenSearch), hunting HELK (Sysmon/Linux/Zeek + Sigma/MITRE), collecte DFIR Velociraptor, timelines Timesketch, stack CTI (OpenCTI, MISP, TheHive, Cortex) et supervision Grafana. Les portails CERT et IT unifient l'accès analyste sans multiplier les URLs.

Modules fonctionnels

Portail CERT

Vue d'ensemble SOC, santé 11+ services, ingest, incidents master, hunting HELK, DFIR VR, CTI, administration. Fichier : portal-cert/server.js.

Portail IT

Dépôt evidences via token CERT, suivi opérations, sync HELK/VR lab. URL : /it/.

Ingestion

Worker Python : EVTX, Plaso, CSV, syslog, STIX → OpenSearch + Timesketch + MinIO. Queue Redis.

OpenSearch

Cluster 2 nœuds, Dashboards, Logstash/Filebeat, indices forensic-*, fp-*, sync HELK/VR.

HELK sidecar

Pipelines Logstash 0010–0099, Kibana hunting, Grafana helk-*, bridge sync/export.

Velociraptor

GUI + bridge collecte, artefacts ForensicFull, playbooks offline lab, export OS/TS/CERT.

Timesketch

Investigation chronologique, import auto ingest, exports HELK/VR.

CTI & IR

OpenCTI hub STIX, MISP IOC, TheHive cas, Cortex analyseurs, webhook portail.

Observabilité

Grafana, Prometheus, Loki — dashboard fp-platform-health-gf, heatmap portail.

Limites ingest (portail)

  • Maximum 100 fichiers par session upload
  • Taille max 5,4 Go par fichier
  • Extensions : EVTX, EVT, LOG, syslog, Plaso, CSV, JSONL, PCAP, STIX, ZIP/KAPE…
  • Option Envoyer vers HELK sur upload CERT pour alimenter le hunting
